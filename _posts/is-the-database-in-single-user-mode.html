title: Is the database in single user mode? date: 2006/05/23 06:58:00 -0700 tags: [{"name":"SQLServer","id":3773161}] author: Jane Dallaway alias: /is-the-database-in-single-user-mode One of the products I help to look after has recently gained its own upgrade process - so you put in the CD and "stuff just happens". As part of this the database gets upgraded, and whilst doing so it is put into single user mode. Unfortunately, one customer has had a problem, and the upgrade has failed. We thought it might be in single user mode, but weren't convinced, so to check we ran


sp_dboption [databasename], 'single user'

which resulted in


OptionName   CurrentSetting
-----------  --------------
single user  on

to change this we simple issued


sp_dboption [databasename], 'single user', false

and checked again and this time it said


OptionName   CurrentSetting
-----------  --------------
single user  off

Success! Now to fix the process...
